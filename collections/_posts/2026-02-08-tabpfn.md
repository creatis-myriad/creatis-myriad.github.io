# Motivations

* Deep learning methods have traditionally struggled with tabular data, because of the heterogeneity between datasets and the heterogeneity of the raw data itself

* Tables contain columns, also called features, with various scales and types (Boolean, categorical, ordinal, integer, floating point), imbalanced or missing data, unimportant features, outliers and so on.

* This made non-deep-learning methods, such as tree-based models, the strongest contender so far

# Mathematical concepts

## Modeling the posterior predictive distribution

* Let's define $$\phi$$ a variable that represents the underlying supervised machine learning task
* The posterior can thus be rewritten as:
  $$p(y_{\text{test}} \mid X_{\text{test}}, X_{\text{train}}, y_{\text{train}})= \int_{\phi}p(y_{\text{test}} \mid X_{\text{test}}, \phi) \, p(\phi \mid X_{\text{train}}, y_{\text{train}}) \, d\phi$$

  * Thanks to Bayes theorm, which says that $$p(A \mid B) = p(B \mid A) p(A) / p(B)$$, this expression can be rewritten as:
  $$p(y_{\text{test}} \mid X_{\text{test}}, X_{\text{train}}, y_{\text{train}}) \propto \int_{\phi}p(y_{\text{test}} \mid X_{\text{test}}, \phi) \, p(X_{\text{train}}, y_{\text{train}} \mid \phi) \, p(\phi) \, d\phi$$ 

  * This formula of the posterior predictive distribution is the theoretical foundation of tabPFN 

<div style="text-align:center">
<img src="/collections/images/tabpfn/tabpfn-posterior-predictive-distribution.jpg" width=700></div>


* The predictive posterior distribution of the target $$y_{\text{test}}$$, given its features $$X_{\text{test}}$$ and a context dataset $$\left(X_{\text{train}}, y_{\text{train}}\right)$$, requires integrating over (latent) tasks

* This integration over tasks has three components:
  * A prior probability $$p(\phi) $$ over tasks, representing how likely a given task is in the integration
  * The likelihood of the training data given the task $$\phi$$, which indicates how likely the observed dataset is for the task over which we integrate
  * The probability of $$y$$ given its features and the task $$\phi$$

> Models like TabPFN approximate this posterior predictive distribution. However, it’s not done by modeling the components of the equation individually, but they take a different approach
